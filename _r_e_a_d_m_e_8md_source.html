4 ## Description
5 The project aims to control a robotic arm with all kind of joystick (home made and commercial ones) or with server web.
6 
7 ## User-space applications
8 
9 The user-space applications can be built using CMake. Some of them have dependencies that you will need to install first: `libevdev-dev libsdl2-dev libudev-dev libboost-dev libmicrohttpd-dev`. I provided a script to build environment by typing this command:
10 
11 ```
12 $ sudo ./get-dependencies.sh
13 ```
14 
15 After installing the packages above, if it exists an error not find directory, find directory `libmicrohttpd-dev.so` in your machine and change `include_directories : /usr/lib/x86_64-linux-gnu/` to your directory.
16 
17 The easiest way to build it is to move to root directory and run `cmake . && make`. This method allows you to build out of source wherever you want.
18 However, in this source directory, a `build.sh .` script is provided for the sake of simplicity.
19 
20 ## Run applications
21 - After compiling source code, you can run program by 2 method:
22 > use play station 3 or nintendo switch
23 ```
24 $ sudo ./armDev --mode "joystick" --config ../config/PS3Joystick.cfg
25 or
26 $ sudo ./armDev --mode "joystick" --config ../config/nintendoJoystick.cfg
27 ```
28 > use web server
29 ```
30 $ sudo ./armDev --mode "server-web" --port [Number of port listening] --standard [http/https]
31 ```

33 ## Run with Docker
34 - I also integrate Docker to this project. This allows users can run easly the application without know which packages supported and how to install them.
35 - First of all, you need to install docker in this link: [https://docs.docker.com/engine/install/ubuntu/](https://docs.docker.com/engine/install/ubuntu/)
36 - You can use dockerfile to generate image and run the container:
37 
38 ```
39 # Build docker image
40 $ docker build -t devminhthuc/controlal5d .
41 # Run container
42 $ docker run -it -p 3000:3000 devminhthuc/controlal5d
43 ```
44 - You can also pull the docker image on my dockerhub from this link:
45 [https://hub.docker.com/r/devminhthuc/controlal5d](https://hub.docker.com/r/devminhthuc/controlal5d)

47 ## Acces service web
48 
49 - open/close connection with robotic arm using `https://localhost:[port]/open`
50 - set position initial using `https://localhost:[port]/initposition`
51 - control robotic arm using example `https://localhost:[port]/shoulder?direction=1`
52 
53 Otherwise, you can control robotic's arm by web application which will call rest api https. This web application give you a joystick virtual.
54 
55 ### Note
56 
57 - The port used to be called in web application is 3000. If you want to call rest api https in another port, please change url fetched in `app.js`
58 - This example call server on the same host machine, so url is `localhost` . if the server is intalled on another machine, you have to modify the source code to url of server in `app.js` and `requestrest.js`. For example, in this source code, you can set the ip address of raspberry PI for: `192.168.43.130`.
59 - To access the server, client needs to connect to the same network as server.
60 - For other options to run docker container, please see in dockerfile to change the execution command `CMD`.
